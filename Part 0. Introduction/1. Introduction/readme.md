- 앱 아키텍처를 설계하는건 매우 중요하다
	- 하지만 체계적이고 세부적인 지식을 얻기 어렵다

- 왜 아키텍처를 배워야 하나?
	- 안드로이드 개발 전반에 가장 큰 영향을 끼치는 것이 앱 아키텍쳐 이다
	- 모든 개발자는 반드시 아키텍처에 대한 깊은 지식을 갖고 있어야 함

- 모바일에만 존재하는 어려운 점이 있다
	- 수많은 이벤트(유저 이벤트, 시스템 이벤트, 라이프사이클 이벤트) 
	- 이벤트 처리 결과도 동시다발적으로 일어날 수 있다
	- 대응이 지연되면 앱이 정지될 수 있다
	-
	- 또한 빠른 배포가 어렵다
	- 심각한 문제를 고치는데에도 시간이 오래걸린다
	-
	- 네트워크가 끊기거나 연결되어 있지 않아도 적절히 동작해야 함	

- 따라서 잘못된 설계를 하면 유지보수성을 극도로 떨어뜨림

- 또한 아키텍쳐는 비즈니스를 촉진시킨다
	- 성공한 앱을 더 빨리 확장시키기 위해 더 많은 모듈화와 유연한 아키텍쳐가 필요함
	- 앱이 더 빨리 솟아 오르려면 새로운 기능에 대한 실험이 빨리 이루어져야 함
	- 더 깊은 사용자화를 하기 위해 유연한 UI 층이 필요하다 

---

- 새로운 아키텍처를 적용하는 2가지 방법
	- 리뉴얼 : 대대적으로 앱을 처음부터 개발하기, 현실적으로 불가능
	- 점진적인 개선 : 매우 큰 리스크를 수반함
	- 어느 쪽이든 아키텍처 지식이 깊어야 함

- 이 강의를 통해
	- 미리 시행착오를 겪을 수 있다
	- 아키텍처와 관련된 의사결정을 위한 중요한 정보를 미리 얻을 수 있다
	- 아키텍처 전반에 걸친 중요한 인사이트를 얻을 수 있다

- 이 뒤는 목차 설명
