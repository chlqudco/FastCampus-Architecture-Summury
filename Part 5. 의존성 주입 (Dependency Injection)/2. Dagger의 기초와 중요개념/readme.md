- 대거란?
	- 코드 생성 기반의 의존성 주입 프레임워크
	- 안드로이드에서는 매우 느린 리플렉션을 사용한 기존 프레임워크보다 안드로이드에 더 적합하지만, 안드로이드 전용은 아님
		- 요즘엔 리플렉션이 매우 느리진 않는데 어쨌든 오버헤드는 있음
		- 전용이 아닌데 웃긴건 다른 곳에서 쓴다는걸 본적은 없음 
	- 안드로이드를 위한 표준 DI 프레임워크
	- 어노테이션 프로세서 기반 코드 생성을 이용해서 구현함
		- 어노테이션 기반이므로 빌드 시간에 상당한 오버헤드 발생
		- 어노테이션 기반이므로 컴파일 에러를 해석하기 굉장히 힘듬
	- 모든 구현이 코드 생성 기반이라 실행 속도가 빠름
	- DI 툴 중 가장 강력하고 유연하며 방대한 양의 기능 제공

- Dagger 주요 개념 : Components
	- Dagger가 객체들의 생성을 관리하기 위해 생성하는 인젝터 클래스의 원형
	- 인터페이스 혹인 abstract 클래스로 정의해야 함
	- Builder / Factory 인터페이스 제공

- Dagger 주요 개념 : Bindings
	- 각 객체의 생성이 실제로 정의되는 곳
	- 함수에 매개 변수가 있는 경우 또 다른 바인딩을 검색해서 주입해줌

- Dagger 주요 개념 : @Inject
	- 간결한 표현을 제공해준다

- 필드 주입 : 의존성 주입을 위한 다른 방법
	- 필드 주입은 생성자를 통하지 않고 주입하는 방법임, 반대 개념은 생성자 주입
	- 내부적으로 같은 패키지 내에 값만 대입하기 위한 클래스를 생성하는 형태로 구현
	- 생성자 주입과는 달리 깔끔한 binding graph가 만들어지기 어려움
	- Dagger가 직접 생성할 수 없는 클래스(액티비티나 프래그먼트)에서 사용됨

- Binding graph
	- 의존성들의 모든 경로를 그래프 형태로 표현한 것

- 충격 발언 : Dagger는 단검아 아니라 DAG(방향 그래프)에서 따온 말임

- Dagger는 그래프를 순회함
	- Dagger가 코드를 처리할 때 component 메소드에서 시작해서 그래프를 완성하게 됨

- Dagger 주요 개념 : Binding Keys
	- 식별자를 key라고 부름
		- Dagger에게 어떤 바인딩이 사용되어야 하는지 쉽게 정의할 수 있음 

- Dagger 주요 개념 : Qualifiers (수식자)
	- 하나의 타입에 대해서 다른 인스턴스를 전달하고 싶은 경우가 있다
	- 타입과 수식자의 조합으로 바인딩 키를 만들 수 있음

- Dagger 주요 개념 : Module
	- 단순히 바인딩들의 모음
	- 멀티모듈의 모듈과는 전혀 다른 개념임
	- 모듈은 컴포넌트 안에서 설치되거나 다른 모듈들 안에 포함됨
	- 주의 : 하나의 모듈이 여러개의 컴포넌트에 설치될 수 없음
	- 주의 : 모듈이 개발자의 실수로 아무곳에도 속하지 않아도 컴파일 에러가 나지 않음

- 멀티 컴포넌트
	- 특정 범위 안에서만 이뤄지는 바인딩이 필요한 경우, Subcomponent를 사용

- 멤버 주입
	- 대부분의 경우 Component에 일일히 getter 메소드를 만드는건 귀찮음
	- 따라서 효율적으로 @Inject를 사용하자
	- Application에서 갖고 있어야 하는 인스턴스들도 @Inject 필드 형태로 만드는게 편함

- Dagger 복습
	- 객체의 생성 혹은 초기화를 담당해줌
		- 개발자가 직접 구현할 필요 없이 대붑누의 코드를 자동으로 생성해줌
	- 어느정도 보일러 플레이트 코드를 제거해줌
		- Dagger 설정을 위한 새로운 보일러 플레이트가 추가되는 측면도 있으나 전체적으로는 감소
	- 시작점으로부터 시작해서 필요한 만큼 탐색하는 형태로 그래프를 분석
	- 대부분의 구현 실수를 컴파일 시에 찾아줌
